Описание классов программы Tracker:

класс Task служит "шаблоном" для новых заявок в трекере (платный?).
класс Bug служит "шаблоном" для заявок по повторному обращению (бесплатно по гарантии?)
класс Item является родительским для Task и Bug, кототые наследуют его поля (могут переопределять методы?)
класс Tracker - хранилище (массив) заявок типа Task и Bug. Должен уметь создавать, редактировать, удалять заявки, выводить список (фильтровать и сортировать?), а также менять статус заявки и добавлять комментарий.
класс Menu - ожидание пользовательского ввода?
